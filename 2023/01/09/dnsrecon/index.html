dnsrecon
查询DNS服务器返回的记录不会触发IDS、IPS等设备即可获取网络设施的相关信息
大部分企业不会监控DNS服务器的，只监控是否存在区域传输漏洞
功能点：
1、区域传输 zone Transfer
2、反向查询 Reverse Lookup
3、暴力猜解 Domain and Host Brute-Foce
4、标准记录枚举 Standard Record Enumeration (wildcard,SOA,MX,A,TXT,etc)
5、缓存窥探 Cache Snooping
6、区域遍历 Zone Walking
7、Google查询 Google Lookup

参数：
-h, –help 帮助信息
-d, –doamin 域名信息
-n, –name_server 要使用的域名服务器 默认使用目标的SOA，多个服务器可以使用 “,” 分隔的列表指定
-r, –range 给定的IP范围进行域名反查 first-last | range |bitmask
-D， –dictionary 用来遍历或者爆破的子域名与主机名的字典文件
-f 保存时 忽略枚举域查找的结果
-a 在标准枚举过程中进行AXFR（完全区域传送）测试
-s 在标准的枚举过程中进行IP地址范围的反向查找
-b 在标准枚举过程中进行bing枚举
-g 在标准枚举过程中进行Google枚举
-y 在标准枚举过程中进行Yandex枚举
-k 在标准枚举过程中进行crt.sh枚举
-w 在标准枚举的过程中进行深度的whois查询和IP反查
-z 在标准枚举的过程中进行DNSSEC域漫游
–threads 指定线程数
–lifetime 指定查询等待的时间
–tcp 使用tcp协议进行查询
–db +filename 将结果存储为sqlite3数据库的文件
–xml +filename 将结果存储为XML格式的文件
-c, –csv +filename 将结果保存到csv格式的文件
-j, –json +filename 将结果保存到json格式的文件
–iw 继续强制执行域，即使发现了通配符记录
-disable_check_recursion 禁用same服务器上的递归查询
–disable_check_bindversion 禁用same服务器上的绑定版本查询
-V 显示dnsrecon的版本
-v 详细的启用信息
-t, –type 指定枚举类型
std 如果NS服务器的域传递失败，进行SOA、NS、A、AAAA、MX 和SRV的枚举（必须使用-d参数指定域名才可使用此参数）
rvl 对给定的IP范围或者CIDR进行反向查找（必须使用-r指定IP范围）
brt 使用指定的字典爆破域名和主机名
srv 枚举srv记录
axfr 对所有的NS服务器进行域传递测试
goo 对子域名和host进行Google搜索
snoop 对 -d 选项给出的文件进行DNS服务器缓存侦测
tld 删除给定域的TLD并测试在IANA中注册的所有TLD
zonewalk 使用NSEC记录进行DNS SEC域漫游
