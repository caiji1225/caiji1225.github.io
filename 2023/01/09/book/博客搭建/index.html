个人博客的搭建框架
1.hexo：https://hexo.io/
Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
Hexo的主题样式也有很多好看的，而且github都有star上万的。

2.jekyll：https://jekyllrb.com/
Jekyll 是一个简单的博客形态的静态站点生产机器。
它有一个模版目录，其中包含原始文本格式的文档，通过一个转换器（如 Markdown）和我们的 Liquid 渲染器转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。
Jekyll 也可以运行在 GitHub Page 上，也就是说，你可以使用 GitHub 的服务来搭建你的项目页面、博客或者网站，而且是完全免费的。

3.hugo：https://www.gohugo.org/
Hugo是一种用Go语言编写的快速，现代的静态网站生成器，旨在让网站创建再次变得有趣。

4.vuepress：https://vuepress.vuejs.org/
VuePress 由两部分组成：一个以 Vue 驱动的主题系统的简约静态网站生成工具，和一个为编写技术文档而优化的默认主题。
它是为了支持 Vue 子项目的文档需求而创建的。
由 VuePress 生成的每个页面，都具有相应的预渲染静态 HTML，它们能提供出色的加载性能，并且对 SEO 友好。
页面加载之后，Vue 就会将这些静态内容，接管为完整的单页面应用程序(SPA)。
当用户在浏览站点时，可以按需加载其他页面。

5.solo：https://solo.b3log.org/
小而美的博客系统，为未来而构建
Solo 是一款小而美的博客系统，专为程序员设计
我们有一个活跃的小众社区，文章可以推送到社区让更多人看到
第一个版本发布于 2010 年，永不断更 !
6.wordpress：https://wordpress.org/
WordPress是使用PHP语言开发的博客平台，用户可以在支持PHP和MySQL数据库的服务器上架设属于自己的网站。
有着许多第三方开发的免费模板，安装方式简单易用。

这里使用
nodejs + hexo + Github OS：本地windows

1、Github 注册一个账户
github命令
echo “# hello” >> README.md
git init
git add README.md
git commit -m “first commit”
git branch -M main
git remote add origin https://github.com/caiji1225/hello.git
git push -u origin main
…or push an existing repository from the command line
git remote add origin https://github.com/caiji1225/hello.git
git branch -M main
git push -u origin main

2、下载Git
1、双击Git.2.39.0.2.exe安装，next
2、路径选择默认或者自定义，next
3、一直默认，直至完成
鼠标右击，选择 git Bash ，输入git 回显即可
4、生成SSH秘钥（id_rsa）和公钥（id_rsa.pub）
ssh-keygen -t rsa #使用RSA算法生成秘钥 默认回车四次
文件路径为：
Linux：~/.ssh
Windows: C:\user\xxx.ssh\

3、Git 绑定Github用户
1、打开github.com 登录
2、点击 用户头像 选择 Settings
3、左侧点击 SSH andGPG keys =》New SSH key =》复制的公钥（id_rsa.pub）内容到Key =》 点击Add SSH Key 完成配置
4、Git Bash 里 执行 ssh -T git@github.com 第一次执行需要输入 yes 确认，出现successfully 即为成功。
5、提交文件（两种方式）
1、本地没有git仓库
1、将远程仓库 clone 到本地
git clone https://github.com/xxx/Demo.git
2、将文件添加并commit到本地仓库
echo “text1234” >text.txt
git status #查看仓库状态
git add text.txt #使用该命令田间道 临时缓冲区
git cimmit -m “备注信息” #提交到本地仓库 #第一次提交 需要配置用户名 和邮箱
git config –global user.mail “caiji1225@163.com“
git config –global user.name “caiji1225”
git log #查看日志
git status #查看状态
3、本地仓库内容push到远程仓库
git push origin master #推送本地仓库到远程仓库
2、本地有git仓库，并且多次commit操作
1、建立一个本地仓库，进入 init进行初始化
cd D:\code\gitbook\Demo
git init
2、关联远程仓库
git remote add origin https://github.com/caiji1225/Demo.git
3、同步远程仓库和本地仓库
git pull origin master
4、本地仓库内容push到远程仓库
git add filename
git commit
git push origin master
4、nodejs
1、下载链接
https://nodejs.org/en/download/
1、双击运行，一直next
2、安装路径默认或者是自定义 ，next （下面的路径是自定义）
3、Add to PATH 选择第一个或者第二个，继续next
4、 直到 Finish
2、测试安装（cmd下）
D:>node -v
v18.12.1
D:>npm -v
8.19.2
3、修改全局设置下载node的缓存等文件（使用默认全局和缓存 可以跳过 该修改和新建步骤）
1、在nodejs 文件夹下新建 node_Cache 、node_Global 两文件夹
2、命令行执行
D:>npm config set prefix “D:\software\nodejs\node_global” #全模块
D:>npm config set cache “D:\software\nodejs\node_cache” #缓存
3、修改系统变量
1、用户变量下的Path里的 npm路径 修改为：D:\software\nodejs\node_global 路径
2、系统变量下新建 NODE_PATH 添加路径为：D:\Software\nodejs\node_global\node_modules
3、系统变量下Path 里 新建添加 D:\Software\nodejs\node_global 路径
4、点击保存
5、验证：npm install express -g #(-g 全局安装的意思，不加默认下载到当前用户变量下的Path定义目录)
6、错误：1、mkdir 修改当前文件夹权限 2、根据错误内容百度即可
4、更换npm源（不更换源，有时候得扶墙）
1、查看初始npm源，如图：
npm config get registry
https://registry.npmjs.org/
2、更换镜像为淘宝镜像
npm config set registry https://registry.npm.taobao.org/
3、检查配置是否成功
npm config get registry
4、安装cnpm （npm在海外，响应慢，cnpm 淘宝提供，同步更新，10分钟延迟，响应快）
1、全局安装基于淘宝源的cnpm
npm install -g cnpm –registry=https://registry.npm.taobao.org
2、下载完后，我们在本地全局文件夹下就能看到cnpm模块
3、执行命令查看cnpm是否回显代表着是否安装成功 或者是在文件夹下能看见该文件夹
cnpm -v

5、通过npm 安装 hexo 博客框架，本地生成博客
1、在安装之前 在github 先建一个仓库 名为：用户名.github.io
2、安装hexo
npm install hexo-cli -g #全局安装Hexo
cd D:\code\gitbook\blog
hexo init blog #初始化博客
npm install
hexo server
hexo g #静态部署 生成静态文件到默认设置的public文件夹(hexo generate 缩写)
hexo d #自动生成网站静态文件，并部署到设定的仓库（dexo deploy 缩写）
hexo clean #清除缓存文件db.json 和已生成的静态文件public
hxo s #查看部署信息
http://localhost:4000 查看你页面信息
3、本地Hexo博客同步到 github
1、回到blog 文件夹下
2、打开 __config.yml 修改配置
原配置
# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
type: ‘’
修改后配置
# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
type: git
repository: https://github.com/caiji1225/caiji1225.github.io.git
branch:master

    3、回到blog文件夹 打开Git Bash
        1、执行：
            npm install hexo-deployer-git --save
        2、分别执行
            hexo clean   #清除缓存文件 db.json 和已生成的静态文件 public
            hexo g       #生成网站静态文件到默认设置的 public 文件夹(hexo generate 的缩写)
            hexo d       #自动生成网站静态文件，并部署到设定的仓库(hexo deploy 的缩写)
        3、打开浏览器，输入 https://xxx.github.io 就可以打开你的网页了：
