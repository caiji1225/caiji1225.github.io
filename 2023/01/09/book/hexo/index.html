hexo 安装后使用

init
hexo init folderName (文件夹名)
新建一个网站。如果没有设置 folderName ，Hexo 默认在目前的文件夹建立网站。
本命令相当于执行了以下几步：Git clone hexo-starter 和 hexo-theme-landscape 主题到当前目录或指定目录。
使用 Yarn 1、pnpm 或 npm 包管理器下载依赖（如有已安装多个，则列在前面的优先）。npm 默认随 Node.js 安装。
new
hexo new [layout] <br> 新建文章，没有设置layout 默认使用_config.yml 中的default_layout 参数代替<br> 如果标题中存在空格，用引号包含<br> hexo new “Use For MS-17-010”<br> 参数<br> -p, –path 自定义新文章的路径<br> -r, –replace 如果文章名已存在，强制替换<br> -s, –slug 文章的Slug，作文新文章的文件名和发布后的URL<br> 默认Hexo 使用文章的标题决定文章文件的路径<br> 独立页面，Hexo会创建以标题命名的目录，在目录中放置index.md<br> 可以使用–path 参数覆盖，从而自定义文件目录<br> hexo new page –path ms17010/use “about use ms17-010”<br> 该命令会创建 source/_posts/ms17010/use.md 同时Front Matter中title指定为”about use ms17-010”<br> hexo new page –path ms17010/use (此命令为错误命令)<br> source/_posts/ms17010/use.md,use.md 的 Front Matter 中的 title 为 “page”<br> 因为在上述命令中，hexo-cli 将 page 视为指定文章的标题、并采用默认的 layout<br>generate<br> hexo generate #生成静态文件（hexo g #缩写）<br> 参数<br> -d, –deploy 文件生成后立即部署网站<br> -w, –watch 监视文件的变动<br> -b, –bail 生成过程若发生未处理的异常，则抛出该异常<br> -f, –force 强制重新生成文件，等价于 hexo clean && hexo generate<br> -c, –concurrency 最大同时生成文件的数量，默认无限制</p> <p>publish<br> hexo publish [layout] <fileName><br> 发表草稿<br>server<br> hexo server<br> 启动服务器<br> 默认建立于4000端口<br> 访问:<a target="_blank" rel="noopener" href="http://localhost:4000/">http://localhost:4000/</a><br> 参数<br> -p, –port 重设端口<br> -s, –static 只使用静态文件<br> -l, –log 启动日记记录，使用覆盖记录格式<br>deploy<br> hexo deploy #(hexo d 缩写)<br> 部署网站<br> 参数<br> -g, –generate 部署之前预先生成静态文件<br>render<br> hexo render <file1> [file2]…<br> 参数<br> -o, –output 设置输出路径<br>migrate<br> hexo migrate <type><br> 从其它博客迁移<br> 参考链接:<a target="_blank" rel="noopener" href="https://hexo.io/zh-cn/docs/migration">https://hexo.io/zh-cn/docs/migration</a><br>clean<br> hexo clean<br> 清除缓存文件(db.json)和静态文件(public)<br> 对主题做修改后,可能需要执行该命令<br>list<br> hexo list <type><br> 列出网站资料<br>version<br> hexo version<br> 显示Hexo版本</p> <p>安全模式<br> hexo –safe<br> 不会载入插件和脚本<br>调试模式<br> hexo –debug<br> 在终端显示调试信息并记录到debug.log<br>简洁模式<br> hexo –silent<br> 隐藏终端信息<br>自定义配置文件路径<br> hexo server –config custom.yml<br> 使用 custom.yml 代替默认的 _config.yml<br>显示草稿<br> hexo –draft<br> 显示 source/_drafts 文件夹中的草根文章<br>自定义CWD (Current Working Directory)<br> 自定义当前工作目录的路径</p> <p>布局（Layout）<br> hexo new [layout] <title><br> 可以在命令中指定文章的布局（layout），默认为 post<br> 可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局<br> Hexo 有三种默认布局：post、page 和 draft, 在创建这三种不同类型的文件时它们将会被保存到不同的路径<br> 而您自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹。<br> 布局 路径<br> post source/_posts<br> page source<br> draft source/_drafts<br> 如果文章想禁用布局,在front-matter 中设置 layout: false<br>文件名称<br> Hexo 默认以标题做为文件名称，但您可编辑 new_post_name 参数来改变默认的文件名称<br> 举例来说，设为 :year-:month-:day-:title.md 可让您更方便的通过日期来管理文章<br> 变量 描述<br> :title 标题（小写，空格将会被替换为短杠）<br> :year 建立的年份，比如， 2015<br> :month 建立的月份（有前导零），比如， 04<br> :i_month 建立的月份（无前导零），比如， 4<br> :day 建立的日期（有前导零），比如， 07<br> :i_day 建立的日期（无前导零），比如， 7<br>草稿<br> Hexo 的一种特殊布局：draft，<br> 这种布局在建立时会被保存到 source/_drafts 文件夹<br> 可通过 publish 命令将草稿移动到 source/_posts 文件夹<br> 该命令的使用方式与 new 十分类似，您也可在命令中指定 layout 来指定布局。<br> hexo publish [layout] <title><br> 草稿默认不会显示在页面中，您可在执行时加上 –draft 参数，或是把 render_drafts 参数设为 true 来预览草稿。<br>模版（Scaffold）<br> 在新建文章时，Hexo 会根据 scaffolds 文件夹内相对应的文件来建立文件，例如：<br> hexo new photo “My Gallery”<br> 在执行这行指令时，Hexo 会尝试在 scaffolds 文件夹中寻找 photo.md，<br> 并根据photo.md的内容建立文章<br> 变量 描述<br> layout 布局<br> title 标题<br> date 文件建立日期 </p> </div> <footer class="article-footer"> <a data-url="http://example.com/2023/01/09/book/hexo/"
