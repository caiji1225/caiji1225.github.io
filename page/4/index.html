Windows 认证
./Kerberos.jpg

认证协议（两种 ）
NTLM （NT LAN Manager） （质询/应答）消息交互模式
用于wind 2000 及之后的 工作组环境

Kerberos	（双向认证）
    用于win 2000 及之后的 域环境
    来源 希腊神话，冥界守护神兽，三头的怪兽，寓意着认证过程涉及三方：客户端、服务端、KDC（Key Distribution Center 秘钥分发中心 ）
    域环境中 KDC 由DC 肩负
    是一种基于票据的认证方式（Ticket）

区别：
Kerberos 更安全、高效，且认证过程更复杂。

Kerberos工作原理 （双向认证模式）

客户端要访问服务器的资源，需先购买服务端认可的票据。等待服务验票之后才能入场。
但是这张票不能直接购买，需要一张认购权证(TGT)。
客户端在买票之前需要预先获得一张认购权证。这张认购权证和进入服务器的入场券均有KDC发售

认购权证（TGT）获得 （步骤1、2）
TGT 属于KDC的认证服务
用户使用域用户登录某主机时，用户本机Kerberos服务向KDC的认证服务发送认证请求（该请求包含两部分：明文形式的用户名、加密用于证明访问者身份的Authenticator）
KDC收到请求后，通过AD（account database）获取该用户信息，通过获取的密码信息生成一个秘钥对Authenticator进行解密，根据解密后的内容和已知是否一致，从而确认登录者身份
KAS成功认证后，生成一个用于确保该用户和KDC通信安全的会话密钥（Logon Session Key），并采用该用户密码派生的秘钥进行加密
接着KAS为该用户创建认购权证 （TGT）（包含两方面：用户相关信息、Logon Session Key），整个TGT通过KDC自己的秘钥进行加密，最终不同秘钥加密的Logon Session Key 和TGT被返回给客户端

通过认购权证（TGT）获得入场卷（步骤3、4）
客户端获得购买进入同域主机入场卷的认购凭证（ TGT），以及Logon Session Key后，缓存本地
凭借TGT向KDC购买入场卷（即服务票据ST：Service Ticket）
客户端向TGS（Ticket Granting Service）发送ST购买请求。（请求包含：客户端用户名；通过Logon Session Key加密的Authenticator；TGT和访问的服务器名）
TGS收到请求后，通过自己的秘钥解密TGT并获取Logon Session Key，通过Logon Session Key解密Authenticator，确认对方身份
身份确认后，TGS生成一个用于客户端和服务器之通信安全的会话密钥（Service Session Key），该会话秘钥通过Logon Session Key加密，
然后出售给客户端需要的入场卷（ST 包含两方面：客户端用户信息和Service Session Key），整个ST通过服务器密码派生的秘钥进行加密
最终两个被加密的Service Session Key 和ST返回给客户端

凭票入场
客户端收到TGS回复后，通过缓存的Logon Seesion Key解密得到Service Session Key，同时得到了服务器入场券ST，且缓存Service Session Key到本地
服务器在收到ST后，确认，予以放行。
如何确认：是服务器收到ST时还附带了一份通过Service Session Key加密的Authenticator
因为ST是通过服务器密码派生的秘钥加密的，所以服务器收到ST时，通过自己的密码派生秘钥进行ST的解密，从中提取Service Session Key
通过提取出来的Service Session Key 进行解密Authenticator。从而验证身份确认ST不是伪造的
服务器对客户端的认证已完成，开始客户端对服务器的认证（双向认证）
服务器将解密后的AUthenticator再次用Service Session Key加密发送给客户端
客户端用缓存的Service Session Key进行解密，和已知做对比，从而证明访问的服务器和自己拥有的Service Session Key 相同（步骤5、6）

例：使用域账户登录域主机，访问另一台域主机，如果对访问和被访问者进行双向身份验证

dnsrecon
查询DNS服务器返回的记录不会触发IDS、IPS等设备即可获取网络设施的相关信息
大部分企业不会监控DNS服务器的，只监控是否存在区域传输漏洞
功能点：
1、区域传输 zone Transfer
2、反向查询 Reverse Lookup
3、暴力猜解 Domain and Host Brute-Foce
4、标准记录枚举 Standard Record Enumeration (wildcard,SOA,MX,A,TXT,etc)
5、缓存窥探 Cache Snooping
6、区域遍历 Zone Walking
7、Google查询 Google Lookup

参数：
-h, –help 帮助信息
-d, –doamin 域名信息
-n, –name_server 要使用的域名服务器 默认使用目标的SOA，多个服务器可以使用 “,” 分隔的列表指定
-r, –range 给定的IP范围进行域名反查 first-last | range |bitmask
-D， –dictionary 用来遍历或者爆破的子域名与主机名的字典文件
-f 保存时 忽略枚举域查找的结果
-a 在标准枚举过程中进行AXFR（完全区域传送）测试
-s 在标准的枚举过程中进行IP地址范围的反向查找
-b 在标准枚举过程中进行bing枚举
-g 在标准枚举过程中进行Google枚举
-y 在标准枚举过程中进行Yandex枚举
-k 在标准枚举过程中进行crt.sh枚举
-w 在标准枚举的过程中进行深度的whois查询和IP反查
-z 在标准枚举的过程中进行DNSSEC域漫游
–threads 指定线程数
–lifetime 指定查询等待的时间
–tcp 使用tcp协议进行查询
–db +filename 将结果存储为sqlite3数据库的文件
–xml +filename 将结果存储为XML格式的文件
-c, –csv +filename 将结果保存到csv格式的文件
-j, –json +filename 将结果保存到json格式的文件
–iw 继续强制执行域，即使发现了通配符记录
-disable_check_recursion 禁用same服务器上的递归查询
–disable_check_bindversion 禁用same服务器上的绑定版本查询
-V 显示dnsrecon的版本
-v 详细的启用信息
-t, –type 指定枚举类型
std 如果NS服务器的域传递失败，进行SOA、NS、A、AAAA、MX 和SRV的枚举（必须使用-d参数指定域名才可使用此参数）
rvl 对给定的IP范围或者CIDR进行反向查找（必须使用-r指定IP范围）
brt 使用指定的字典爆破域名和主机名
srv 枚举srv记录
axfr 对所有的NS服务器进行域传递测试
goo 对子域名和host进行Google搜索
snoop 对 -d 选项给出的文件进行DNS服务器缓存侦测
tld 删除给定域的TLD并测试在IANA中注册的所有TLD
zonewalk 使用NSEC记录进行DNS SEC域漫游

dnsenum nzsis.govt.nz

A 记录 表示域名或者主机名对应的IPV4的记录
NS 记录 指定了子域对应的域名服务器
MX 记录 是域名与邮件交换服务器的对应记录

此命令会尝试从名字服务器进行完全区域传输，但可能会因为名字服务器对区域传输做限制导致失败
在 ：
Trying zone Transfers and getting Bind Versions：

显示的内容就是。

区域传送： DNS分为 主、从服务器。从服务器向主服务器查询更新信息，主服务器将本级的区域文件的数据传递给从服务器，此过程称为区域传送
区域传送分为
完全区域传送 （AFXR） 和增量区域传送 （IXFR）
AFXR：主服务器将区域文件中的所有数据传递给从服务器
IXFR：主服务器将区域文件中更新的部分传递给从服务器

DNS

DNS
DNS协议是一种请求/应答协议，属于应用层的隧道
大量的防御设备的使用导致，传统的socket隧道濒临淘汰，TCP、UDP被拦截
DNS、ICMP、HTTP|HTTPS等协议成为主流
DNS报文具有穿透防火墙的能力，且大量设备不会拦截过滤DNS流量。
DNS协议常用于C&C通信
C&C服务器（Command and Control Server，命令及控制服务器）
C&C节点分为两种 服务端（属于攻击者） 客户端（被控者）

原理
将其它协议封装在DNS协议中进行传输，通过逐层的DNS服务器进行查询和中转

IPC 入侵

IPC$
IPC$(Internet Process Connection)可以被理解为一种“专用管道”.可以在连接双方建立一条安全的通道，实现对远程计算机的访问。
Windows NT/2000/XP提供了IPC$功能的同时，在初次安装系统时还打开了默认共享，即所有的逻辑共享(C$,D$,E$……)和系统目录(ADMIN$)共享。
所有这些共享的目的，都是为了方便管理员的管理，但在有意无意中，导致了系统安全性的隐患
Windows XP安装后的默认共享方式就是”仅来宾”的方式。
利用IPC$,连接者甚至可以与目标主机建立一个空的连接而无需用户名与密码(对方机器开了ipc$共享,否则连接不上的)
而利用这个空的连接，连接者还可以得到目标主机上的用户列表(管理员会禁止导出用户列表的)。
以空连接登陆时,没有任何权限,以用户或管理员的身份登陆时,你就会有相应的权限

错误号 5 ，拒绝访问：很可能你使用的用户不是管理员权限的；
错误号 51 ， Windows 无法找到网络路径：网络有问题；
错误号 53 ，找不到网络路径： ip 地址错误；目标未开机；目标 lanmanserver 服务未启动；目标有防火墙（端口过滤）；
错误号 67 ，找不到网络名：你的 lanmanworkstation 服务未启动或者目标删除了 ipc$ ；
错误号 1219 ，提供的凭据与已存在的凭据集冲突：你已经和对方建立了一个 ipc$ ，请删除再连；
错误号 1326 ，未知的用户名或错误密码：原因很明显了；
错误号 1792 ，试图登录，但是网络登录服务没有启动：目标 NetLogon 服务未启动；
错误号 2242 ，此用户的密码已经过期：目标有帐号策略，强制定期要求更改密码。

查看本地共享
net share

删除共享
net share C$ /del

关闭共享
net stop server /y (重启会自动开启)
建立空连接
net use \10.61.10.130\ipc$ /user:”” “”

建立IPC$ 连接
net use \10.61.10.130\ipc$ “” /user:””
查看远程主机的时间
net time \10.61.10.130

查看远程主机的共享服务 (看不见默认共享的)
net view \10.61.10.130

远程主机映射
net user z: \10.61.10.130\c$
成功后 你的Z盘就是130主机的C盘

删除映射盘
net use z: /del
